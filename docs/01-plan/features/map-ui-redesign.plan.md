# Plan: 지도 기반 UI 전면 재설계

- **프로젝트명:** NaverLand Web Dashboard v2 (Map-centric)
- **참고:** https://estate-rader.com/
- **작성일:** 2026-03-16
- **상태:** Draft

---

## 1. 현황 분석 & 문제점

### 현재 문제
1. **검색 결과가 테이블만** - 지도 없이 텍스트 목록만 표시
2. **대시보드 미동작** - Supabase 데이터는 있으나 차트가 빈 상태
3. **UX 부족** - 단순 CRUD 느낌, 부동산 서비스답지 않음
4. **시각적 임팩트 없음** - 카드/차트/지도 없음

### estate-rader.com 벤치마크
- **지도 중심 레이아웃**: Kakao Maps + 클러스터 마커
- **실시간 통계 카드**: 급매수, 신규, 가격인하 건수
- **리더보드**: 단지별/지역별 TOP 10
- **다중 필터**: 매물유형 탭 + 지역 + 가격 비율 필터
- **AI 점수**: 급매 점수 시각화

## 2. 재설계 목표

### 2.1 핵심 컨셉
> **"지도에서 매물을 한눈에, 클릭하면 상세까지"**

### 2.2 주요 변경
| 현재 | 변경 후 |
|------|--------|
| 검색 → 테이블 결과 | **지도 + 사이드 패널** |
| 텍스트 필터 | **지도 위 필터 오버레이** |
| 빈 대시보드 | **통계 카드 + 실제 차트** |
| 별도 대시보드 페이지 | **메인 화면에 통합** |

## 4. 기술 스택 변경

| 항목 | 현재 | 변경 |
|------|------|------|
| 지도 | 없음 | **Kakao Maps SDK** (무료, 클러스터러 포함) |
| UI | 인라인 Tailwind | **shadcn/ui 컴포넌트** 활용 강화 |
| 레이아웃 | 단순 페이지 | **사이드바+지도 Split View** |
| 차트 | Recharts (미동작) | **Recharts 수정 + 실데이터 연결** |

### Kakao Maps 설정
- App Key: 사용자가 https://developers.kakao.com 에서 발급
- 라이브러리: `clusterer` (클러스터 마커)
- 무료 티어: 30만 호출/일 (충분)

## 5. 구현 순서

### Phase 1: 지도 기반 메인 화면 (핵심)
1. Kakao Maps 연동 (`react-kakao-maps-sdk`)
2. 사이드바 + 지도 Split Layout
3. 매물 마커 표시 (가격 라벨)
4. 클러스터링 (구 단위 → 개별 매물)
5. 마커 클릭 → 상세 팝업
6. 사이드바 매물 목록 ↔ 지도 연동 (클릭 시 이동)

### Phase 2: 대시보드 수정
7. 대시보드 API → DB 실데이터 연결 확인/수정
8. 통계 카드 실데이터
9. 차트 실데이터 바인딩
10. 최근 변동 매물 TOP 목록

### Phase 3: 부가 기능
11. 자연어 검색 → 지도 자동 이동
12. 필터 변경 → 지도 실시간 갱신
13. 반응형 (모바일: 지도 + 하단 시트)
